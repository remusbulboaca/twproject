Abstract

Sa se realize o aplicatie Web destinata colectionarilor de capace, dopuri si etichete specifice recipientelor folosite pentru inmagazinarea bauturilor (ne)alcoolice. Pe baza facilitatilor de cautare multi-criteriala implementate, utilizatorii autentificati vor putea crea, inventaria si partaja informatii privind artefactele detinute in functie de diverse caracteristici: atribute fizice, imagine, valoare, tara de provenienta, istoric al folosirii efective etc. Se va oferi suport pentru import/export de date. Se va realiza, de asemenea, un clasament al celor mai populare obiecte colectate, disponibil si ca flux de date RSS.

Introduction

COLR este o platforma in care useri se pot inregistra si autentifica pentru gestionarea colectiilor de capace. La inregistrare useri vor furniza site-ului informatii precum numele, prenumele, email, username (unic), parola. Site-ul verifica username-ul si email-ul sa fie unic in baza de date. In caz ca exista deja un user cu aceste campuri site-ul afiseaza eroarea respectiva. Toate campurile sunt obligatorii iar fiecarui utilizator ii corespunde un id generat automat. De asemenea parola este criptata in format MD5. Un cont nou trebuie activat folosind un link trimis pe email. Codul de verificare este unic generat. In caz ca un utilizator si-a uitat parola se va genera un token pentru securitate dar si un cookie pentru a avea doar 5 minute la dispozitie. Daca email-ul este corect si apartine unui user din baza de date acesta va prima pe email un link pentru recuperarea perolei. Dupa 5 minute linkul expira si trebuie generat altul. La autentificare este posibila functia de remember me ce permite user-ului sa ramana autentificat chiar si la inchiderea browserului. Daca este gresit username-ul sau parola platforma afiseaza eroarea.

Adaugarea unui produs se face printr-un formular in care user-ul completeaza numele, anul, descrierea, istoria si locatia produsului. Locatia este verificata sa fie una valida. Produsul este trimis catre Admin Panel in sectiunea pending pentru a fi aprobat de catre un admin. Astfel, ne concentram pe corectitudinea si integritatea produselor postate. Fiecare imagine urcata are o denumire unica pentru nu a exista eventuale probleme legate de suprascriere a fisierelor

Pagina produselor contin detalii despre acestea preluate din baza de date precum si o harta cu locatia. Din aceasta pagina se poate accesa pagina de search, profilul userului ce are capacul dar si profilul personal. De asemenea daca produsul apartine userului autentifict acesta poate fi sters simultan din baza de date a produselor si a favoritelor. Produsul poate fi adaugat la favorite de catre orice user sau scos.

Pagina de search afiseaza in functie de cautare capacele si useri. Acestea sunt generate automat prin interogarea bazei de date. Se pot sorta dupa an, locatie dar si data la care au fost adaugate.

Pagina de profil contine :

  • Vizualizarea detaliilor persoanle pentru a contacta userul
  • Vizualizarea produselor personale generate automat din baza de date
  • Vizualizarea produselor pe lista de favorite

Pe pagina de profil personal sunt de asemenea adaugate optiunea de logout, schimbare a pozei de profil, stergerea contului definitiv dar si resetare a parolei.

Users DB

In tabela users sunt stocate toate informatiile despre useri inregistrati pe platforma. Parola este criptata MD5 iar validation code reprezinta codul generat unic pentru activarea contului si resetarea parolei. Admin Panel

Admin panelul este folosit pentru a controla useri si produsele de pe site. Pentru accesul la acest panou de comanda este necesar un cont de administrator. Admin Panelul are 4 sectiuni: dashboardul, useri,produse si baza de date.

Dashboard

In Dashboard sunt statistici referitoare la numarul de useri inregistrati, numarul de produse totale si cate produse au fost adaugate astazi

Members

Categoria de membri este impartita si ea la randul ei in 4 subcategorii: Useri, Admini, Adaugarea unui user fara a mai fi necesara trimiterea unui mail si adaugarea unor useri normali privilegiile de admin. Pentru fiecare subsectiune, administratorii au acces la toata baza de date cu userii. Am implementat o functie de search folosind XMLHttpRequest pentru a fi in timp real iar de asemenea tabela este pagina in orice moment folosind un script scris in Javascript.

Panoul de add admin este accesibil pentru toata lumea dar se poate folosi doar de catre un utilizator cu atributia de GM. Acelasi lucru este valabil si la functia de desetare a unui admin de la privilegii

Products

Valabil si ca la panorile de membri, aceasta este structurata in 3 parti: vizualizarea listei de produse, listei de favorite dar si listei de produse in asteptare. Toate tabelele sunt paginate si au implementata functia de search folosind Ajax.
